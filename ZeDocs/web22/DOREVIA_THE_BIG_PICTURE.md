# The Big Picture — DIVA & DLP
## Gouvernance par la preuve et mémoire des intentions

Version : v1.0  
Statut : Concept fondateur validé  
Portée : Vision produit (non technique)  
Complément technique : `VISION_TECHNIQUE_DOREVIA_v1.0.md`  

---

# 1. Problématique

Les entreprises pilotent leurs activités à partir de chiffres : trésorerie, revenus, charges, fiscalité, performance.

Ces données décrivent des faits économiques.

Cependant, les chiffres seuls ne permettent pas de comprendre :

- Pourquoi une décision a été prise
- Quelle intention stratégique l’a motivée
- À quel moment précis elle s’inscrit
- Quel impact était attendu

Avec le temps, les indicateurs évoluent.
Mais l’intention initiale disparaît.

Il manque un lien structuré entre :

> Les faits économiques observés  
et  
> Les intentions de gouvernance qui les ont précédés.

---

# 2. Le socle : la vérité scellée

Dorevia repose sur un principe fondamental :

Les données financières sont scellées, horodatées et vérifiables.

Elles constituent la base factuelle unique.

DIVA lit exclusivement :

- Les données issues du Vault
- Les agrégats validés
- Les snapshots horodatés
- Les événements probants

Aucune décision n’est générée à partir de données non scellées.

La vérité économique est stable, traçable et juridiquement défendable.

---

# 3. Le rôle de DIVA

DIVA n’est pas décisionnaire.

DIVA est un module de :

- Lecture
- Synthèse
- Qualification
- Formalisation

Elle :

- Résume la situation contextuelle à partir des données scellées
- Rend la structure économique lisible
- Analyse un avis pour action saisi par un utilisateur
- Vérifie :
  - Pertinence professionnelle
  - Impact potentiel sur l’entreprise
  - Conformité éthique

Elle ne :

- Propose pas spontanément des décisions
- N’invente pas d’intention
- Ne remplace pas le jugement humain
- Ne modifie pas les données

Son rôle est un rôle de traduction et de structuration.

---

# 4. L’avis pour action (humain)

L’utilisateur exprime librement une intention.

Cette intention peut concerner :

- Une décision financière
- Une initiative organisationnelle
- Une action stratégique
- Une orientation RH
- Toute décision ayant un impact potentiel sur l’entreprise

L’utilisateur ne peut pas créer directement une DLP.

Il exprime une intention.  
DIVA formalise.

---

# 5. La DLP : marqueur d’intention

Une DLP est :

> Un marqueur d’intention daté, rattaché à un contexte économique donné, et susceptible d’expliquer ultérieurement une évolution des indicateurs.

Ce n’est pas :

- Une tâche opérationnelle
- Une note libre
- Une suggestion IA
- Une action exécutée

C’est :

Un repère stratégique dans le temps.

La DLP capture :

- L’intention
- Le moment
- Le contexte
- L’impact attendu

Elle constitue un point de référence explicatif.

---

# 6. Génération des DLP

Lorsqu’un avis pour action est soumis :

DIVA vérifie :

1. Caractère professionnel
2. Impact potentiel sur l’entreprise
3. Conformité éthique
4. Cohérence minimale

Si les critères sont remplis :

- DIVA génère 5 DLP structurées
- L’utilisateur valide ou rejette
- Les DLP validées sont archivées

Si les critères ne sont pas remplis :

- Aucune DLP n’est générée
- DIVA explique factuellement pourquoi

Aucune DLP libre n’est autorisée.

---

# 7. Structure d’une DLP

Chaque DLP contient :

- Date
- Énoncé synthétique causal
- Catégorie d’impact (coûts, revenus, organisation, fiscalité, performance…)
- Référence au snapshot financier
- Responsable

Deux niveaux d’affichage :

- Version longue : registre probant complet
- Version courte : fil d’actualité synthétique

---

# 8. La chaîne complète

Vérité scellée  
→ Résumé contextuel (DIVA)  
→ Avis humain  
→ Contrôle IA  
→ Génération DLP  
→ Validation humaine  
→ Archivage probant  
→ Observation d’impact dans le temps  

Cette chaîne relie :

Les faits économiques  
aux intentions stratégiques  
dans une continuité traçable.

---

# 9. Ce que cela crée

Dorevia ne se limite pas à un cockpit financier.

Le système crée :

- Une mémoire stratégique structurée
- Un registre des intentions
- Une traçabilité des décisions
- Une capacité d’explication a posteriori

Les chiffres montrent ce qui s’est produit.  
Les DLP expliquent pourquoi cela a été voulu.

L’IA éclaire.  
L’humain décide.  
La preuve structure le tout.

---

# 10. Positionnement fondamental

DIVA n’est pas un assistant décisionnel.

DIVA est un instrument de gouvernance par la preuve.

La DLP n’est pas une tâche.

C’est un marqueur d’intention inscrit dans l’histoire économique de l’entreprise.
