# 📊 Analyse UX — SPEC HERO v2.0 Dorevia-Vault

**Date** : 2026-01-20  
**Analyste** : UX Designer  
**Version analysée** : v2.0  
**Statut** : Analyse critique + Recommandations

---

## 🎯 Vue d'ensemble

### Points forts ✅

1. **Objectif clair** : "Compris en < 5 secondes" — excellent point de départ
2. **Cible bien définie** : Entrepreneurs (indépendants, PME, ETI)
3. **Ton approprié** : Simple, concret, terrain — aligné avec la cible
4. **Structure hiérarchique** : Logique et progressive
5. **Interdits clairs** : Évite le jargon fintech — bonne pratique

---

## ⚠️ Problèmes identifiés (P0-P2)

### 🔴 P0 — Critique : Manque de différenciation

**Problème** :
- Headline "Dorevia-Vault, votre assistant financier" = **trop générique**
- Sous-titre "Pour suivre votre trésorerie et vos chiffres clés" = **pourrait être n'importe quel dashboard**
- Risque de confusion avec des outils comme : Compta-Facile, Sage, QuickBooks, etc.

**Impact** : 
- L'utilisateur ne comprend pas **ce qui différencie** Dorevia-Vault
- La proposition de valeur unique (preuve, sécurisation, opposabilité) est **noyée**

**Recommandation** :
```
Headline (option 1 — orienté résultat) :
"Vos données financières deviennent des preuves vérifiables."

Headline (option 2 — orienté usage) :
"Pilotez votre trésorerie avec des données certifiées."

Sous-titre :
"Chaque opération validée dans votre ERP devient automatiquement 
une preuve opposable, sans rien changer à vos habitudes."
```

---

### 🟠 P1 — Important : Bloc descriptif trop dense

**Problème** :
- 3 messages mélangés dans un seul bloc :
  1. Cible (indépendant → ETI)
  2. Simplicité
  3. ERP open-source
- Risque de surcharge cognitive

**Impact** : Ralentit la compréhension rapide (< 5 secondes)

**Recommandation** :
```
Option A — Scinder en 2 blocs :
1. Bloc cible (court) :
   "Pour toutes les entreprises, de l'indépendant à l'ETI."

2. Bloc différenciation (sous les bullets) :
   "La solution la plus simple pour sécuriser vos données financières 
   lorsque vous utilisez un ERP open-source."

Option B — Simplifier en 1 phrase :
"Pour toutes les entreprises qui utilisent un ERP open-source."
```

---

### 🟡 P2 — Amélioration : Phrase signature sous-utilisée

**Problème** :
- "De la vente à la banque, chaque décision devient une preuve" = **très fort**
- Placée "sous le hero ou en micro-texte" = **trop discret**
- C'est pourtant le vrai différenciateur !

**Impact** : Le message le plus différenciant est relégué en second plan

**Recommandation** :
```
Option 1 — Intégrer dans le sous-titre :
"De la vente à la banque, chaque décision devient une preuve. 
Dorevia-Vault transforme vos opérations en données certifiées."

Option 2 — Badge ou tagline visible :
Badge : "Infrastructure souveraine de vérité financière"
+ Sous-titre : "De la vente à la banque, chaque décision devient une preuve."
```

## 📝 Recommandations spécifiques

### 1. Headline — Prioriser la différenciation

**Actuel** :
> Dorevia-Vault, votre assistant financier.

**Problème** : Trop générique, ne dit pas "pourquoi Dorevia-Vault"

**Alternatives proposées** :

**A) Orienté résultat (recommandé)** :
```
"Vos données financières deviennent des preuves vérifiables."
```
✅ Avantages :
- Différenciation immédiate
- Compréhension rapide
- Aligné avec la vision "preuve"

**B) Orienté usage** :
```
"Pilotez votre trésorerie avec des données certifiées."
```
✅ Avantages :
- Concret, orienté action
- Garde l'aspect "assistant financier" mais avec différenciation

**C) Orienté problème** :
```
"Vos factures deviennent des preuves opposables, automatiquement."
```
✅ Avantages :
- Très concret
- Met l'accent sur l'automatisation

---

### 2. Sous-titre — Clarifier la promesse

**Actuel** :
> Pour suivre votre trésorerie et vos chiffres clés en temps réel.

**Problème** : Pourrait être n'importe quel outil de gestion

**Recommandation** :
```
"Chaque opération validée dans votre ERP devient automatiquement 
une preuve opposable, sans rien changer à vos habitudes."
```

**Alternative (plus court)** :
```
"Transformez vos opérations en preuves vérifiables, en temps réel."
```

---

### 3. Bloc descriptif — Simplifier

**Actuel** :
> Dorevia-Vault s'adresse à toutes les entreprises, de l'indépendant à l'ETI.  
> La solution la plus simple pour sécuriser vos données financières lorsque vous utilisez un ERP open-source.

**Problème** : Trop long, 2 messages distincts

**Recommandation simplifiée** :
```
"Pour toutes les entreprises qui utilisent un ERP open-source."
```

**Ou scinder** :
```
Bloc 1 (cible) :
"Pour toutes les entreprises, de l'indépendant à l'ETI."

Bloc 2 (technique, sous les bullets) :
"Compatible avec les ERP open-source (Odoo, etc.)"
```

---

### 4. Bullets — Aligner avec le message

**Actuel** :
- Indicateurs financiers en temps réel
- Zéro manipulation humaine
- Aligné LNE2026 / NF525

**Analyse** :
- ✅ "Zéro manipulation humaine" = excellent, différenciant
- ⚠️ "Indicateurs financiers en temps réel" = générique
- ⚠️ "Aligné LNE2026 / NF525" = technique, peut-être trop tôt

**Recommandation** :
```
- Preuves vérifiables en temps réel
- Zéro manipulation humaine
- Compatible ERP open-source
```

**Alternative (plus orienté bénéfice)** :
```
- Données certifiées automatiquement
- Zéro manipulation humaine
- Prêt en cas de contrôle
```

---

### 5. Phrase signature — Valoriser

**Actuel** : Placée "sous le hero ou en micro-texte"

**Recommandation** : **Intégrer dans le Hero**

**Option A — Dans le sous-titre** :
```
"De la vente à la banque, chaque décision devient une preuve. 
Dorevia-Vault transforme vos opérations en données certifiées."
```

**Option B — Badge + sous-titre** :
```
Badge : "🇫🇷 Infrastructure souveraine de vérité financière"

Sous-titre : 
"De la vente à la banque, chaque décision devient une preuve."
```
